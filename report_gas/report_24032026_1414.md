### Fix #2 — Kimi Fallback Response Normalization ✅
**File dimodifikasi**: `gas-web-backend/src/api/v1/analysis.py`

```python
# SEBELUM (BROKEN):
data = await _kimi_fallback(feature, payload)
# → Kimi returns {result: {summary: "..."}, source: "kimi-ai-fallback"}
# → Frontend expects {signal, probability, entry, sl, tp1} for "signal" feature
# → KeyError / undefined in frontend

# SESUDAH (FIXED):
data = await _kimi_fallback(feature, payload)
data = _normalize_result(feature, data)  # inject feature-specific fields
```

Fixed di 3 locations:
- `_call_core_with_credits()` (all features)
- `generate_signal()` (signal endpoint)
- `multi_symbol_scanner()` (scanner endpoint)

### Fix #3 — No Double Regime Call in Orchestrator ✅
**File dimodifikasi**: `gas-quant-orch/src/core/orchestrator.py`

```
SEBELUM (BROKEN):
  Phase 1: fetch_regime() → timeout 0.5s → regime="TRENDING" ✓
  selected = ["regime", "trend", "pattern", "orderflow"]
  Phase 2: _run_engines() with "regime" in list → fetch_regime() LAGI! ❌
  Total: 2x HTTP call ke regime-detector per request

SESUDAH (FIXED):
  Phase 1: fetch_regime() → cached_regime_result = {...}
  Phase 2: _run_engines(..., cached_regime=cached_regime_result)
           → if cached_regime is not None: skip fetch, inject directly
  engines_run: ["regime(cached)", "trend", "pattern", "orderflow"]
  Total: 1x HTTP call — hemat 1 upstream call per analyze request
```

### Fix #6 — Tiered ComputeLevel (FAST/STANDARD/FULL) ✅
**Files dimodifikasi**: `gas-strategy-core/src/main.py`, `gas-strategy-core/src/indicators/technical.py`

```python
# SEBELUM: pure_compute=True → skip LLM, tapi SMC selalu dipanggil
# → Ambiguous: "pure compute" tapi masih ada SMC HTTP call

# SESUDAH: compute_level dengan 3 tier:
# FAST     = indicators only (no SMC, no LLM) → ~50ms, pure local compute
# STANDARD = indicators + SMC (no LLM) → ~200ms, backward compat dengan pure_compute=True
# FULL     = indicators + SMC + LLM → ~2-10s, full AI response

TechnicalAnalysisRequest:
  pure_compute: bool = False   # deprecated alias → maps to STANDARD
  compute_level: str = None    # FAST | STANDARD | FULL

Precedence: compute_level > pure_compute
```

Tambahan `compute_indicators_only()` di `technical.py`:
- Pure sync, no async/await
- No SMC engine call (no HTTP)
- Returns same shape as `_build_feature_summary` (termasuk confluence_score, setup_type, SMC=None)

Response sekarang include `compute_level` field untuk debugging/logging.

### Fix #7 — Screener Shared Cache + Distributed Lock ✅
**File dimodifikasi**: `gas-terminal-backend/src/routes/screener.py`

```
Pattern: Anti-thundering-herd dengan Redis distributed lock

SEBELUM: N concurrent users → N parallel POST ke gas-screener-service
         → Screener kelebihan beban, rate limit, atau memori spike

SESUDAH:
  Cache hit  → return langsung (<1ms)
  Cache miss → coba acquire Redis lock (NX flag, TTL 20s)
    ├── Lock acquired (first request):
    │     → call screener, cache result (TTL 30s), release lock
    └── Lock not acquired (concurrent request):
          → poll 300ms intervals sampai cache muncul (max 15s)
          → kalau lock holder crash: retry call sendiri
```

Keys:
- `screener:cache:{sha256[:16]}` — shared result cache, TTL 30s
- `screener:lock:{sha256[:16]}`  — distributed lock, TTL 20s (auto-expire)

Cache key di-hash dari request body (sort_keys=True) — identical request = same cache.
