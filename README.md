## 📊 Observability & Monitoring

Ekosistem GAS dilengkapi dengan stack observabilitas penuh untuk pemantauan real-time.

| Service | Port | Kegunaan | URL Akses (Lokal) |
|---------|------|----------|-------------------|
| **Grafana** | 3000 | Visualisasi & Dashboard | http://localhost:3000 |
| **Loki** | 3100 | Agregasi Log (seperti ELK) | http://localhost:3100 |
| **Prometheus**| 9090 | Metrics & Time-series data | http://localhost:9090 |
| **MinIO** | 9010 | Console (Object Storage) | http://localhost:9010 |

### Cara Monitoring Cepet:
- **Cek Log**: Masuk ke Grafana -> Explore -> Pilih Datasource **Loki** -> Masukkan query `{container=~"gas-.*"}`.
- **Dashboard**: Gunakan dashboard **"GAS Services Overview"** untuk melihat ringkasan status service.
- **Login Grafana**: `admin` / `gas_admin_2026`.

## 🔐 Security Architecture

Sistem keamanan menggunakan model **Defense in Depth**:

1.  **Gateway Auth (Layer 1)**: `gas-gateway-api` memvalidasi JWT (Supabase) untuk request eksternal.
2.  **Internal Security (Layer 2)**: Setiap service memvalidasi `X-Internal-Key` untuk mencegah bypass gateway.
3.  **User Propagation**: Gateway secara otomatis menyuntikkan `X-User-ID` hasil dekripsi JWT agar service internal tahu siapa yang memanggil.

### Contoh Header Internal:
```http
X-Internal-Key: <GATEWAY_API_KEY>
X-User-ID: <UUID-USER>
```
